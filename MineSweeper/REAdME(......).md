设计报告开发与使用部分



开发环境



（1）操作系统：WINDOWS10/11系统

（2）编程语言：C++

（3）开发环境：CLion 2025.3.1 CMake (基于编译参数 -static-libgcc 等判断，搭配 MinGW)



算法实现核心代码/使用的关键技术



1.Win32 API (GUI 与 GDI): 使用原生 Windows API 创建窗口、按钮，处理鼠标/键盘消息 (WndProc)，以及使用 GDI 进行界面的绘制（文本、矩阵、颜色填充、图像拉伸等）。

2.stb\_image.h 库: 一个轻量级的开源单头文件图像加载库，用于读取本地的 .jpg 和 .png 格式图片并转换为位图数据。

3.C 标准文件 I/O: 使用 fopen、fwrite、fread 以二进制形式保存和读取 save.dat 结构体数据，实现游戏存档与读档。



功能模块设计与算法实现



一．功能模块设计



1\. 游戏状态管理模块

&#x20;  使用状态机（"GameState" 枚举）控制游戏流程：

&#x20;  1)STATE\_COVER：初始封面视图，点击进入下一阶段。

&#x20;  2)STATE\_DIFFICULTY：难度选择视图，提供初、中、高级及读档按钮。

&#x20;  3)STATE\_PLAYING：实际游玩视图，渲染棋盘、侧边栏按钮并响应操作。



2\. UI 与图形渲染模块

&#x20;  1)图像加载：使用"stb\_image.h"将"png/jpg"转码为像素数组，并通过 "CreateDIBSection" 转换为原生 Win32 "HBITMAP" 以供绘制。

&#x20;  2)界面绘制：在 "WM\_PAINT" 消息中使用原生 GDI 函数（"FillRect", "FrameRect", "StretchBlt", "DrawText"）绘制背景、方块、地雷、旗帜图片以及数字文本。



3\. 用户交互响应模块

&#x20;  1)鼠标事件：监听 "WM\_LBUTTONDOWN"（左键翻开格子）、"WM\_RBUTTONDOWN"（右键插旗/取消插旗）。

&#x20;  2)控件事件：监听 "WM\_COMMAND" 捕捉并处理右侧菜单及难度选择面版上的按钮点击事件。



4\. 数据持久化模块（存档/读档）

&#x20;  序列化与反序列化：通过 "fwrite" 和 "fread" 将当前的行列数、雷数、游戏状态（输赢状态）以及整个棋盘结构体数组（"g\_board"）直接以二进制形式写入/读取 "save.dat" 本地文件。



二．算法实现



1\. 地雷随机布设机制

&#x20;  使用 "rand()" 函数结合 "while" 循环随机生成坐标 "(r, c)"，如果该位置还不是地雷，则将其标记为地雷，直至达到当前难度要求的总雷数（"g\_numMines"）。



2\. 周围雷数计算算法

&#x20;  对整个棋盘进行双重遍历。当遇到非雷区域时，使用范围为 "\[-1, 1]" 的双重循环扫描其周围的 8 个直接相邻格子。在通过越界检查后，统计含有地雷的格子数量，并将结果缓存到 "neighborMines" 属性中。



3\. 空白区域连续展开算法（Flood Fill 泛洪算法）

&#x20;  在 "RevealCell" 函数中使用\*\*深度优先搜索（DFS）\*\*的思想实现递归展开：

&#x20;  1）终止条件：坐标越界、已翻开、已标记（插旗）或是点中地雷。

&#x20;  2）递归扩散：如果当前格子安全且周围雷数 "neighborMines == 0"，则对周围 8 个方向的相邻格子递归调用 "RevealCell"，从而实现点击一次翻开一大片空白区域的效果。



4\. 胜利判定算法

&#x20;  在每次成功翻开安全格子后触发 "CheckWinCondition"，遍历全局统计已翻开的无雷格子数量，当该数量等于 "总格子数 - 地雷总数" 时，判定游戏胜利。





安装及使用



一．安装环境要求



1.受众定位：面向 Windows 桌面平台。

2.操作系统：Windows 7 / 10 / 11。

3.依赖文件：可执行文件与必备图像资源文件（"start.jpg"、"Background.png"、"flag.png"、"mine.png"）必须存放于同级目录中。

4.开发/编译要求：若需重新编译，需使用支持 C/C++ 的编译器（如 MinGW），配合 CMake 工具链，并在编译参数中开启静态链接选项（如 "-static-libgcc -static"），以此保证最终产物在未安装编译器的设备上正常运行。



二．安装过程



1.玩家部署（默认安装步骤）：

&#x20; 本程序为完全依靠本地文件 I/O 与 Win32 API 运行的便携式（绿色免安装）软件。玩家无需执行传统的应用安装向导：

&#x20; 1)获取包含执行文件与素材的整体压缩包。

&#x20; 2.)将压缩文件解压至硬盘上的任意纯英文或常规路径文件夹中。

&#x20; 3)双击对应编译输出的 ".exe" 二进制文件即可直接运行。

2\. 开发者构建：在 CLion 中打开项目，点击构建按钮生执行文件。由于代码中没有使用绝对路径，务必将构建生成的 ".exe" 拷贝到同图片资源一致的目录进行调测与分发。



三．典型使用流程



该流程涵盖了启动、菜单交互及实际操作的完整主干：



1\. 启动阶段：双击 ".exe" 文件，程序初始化并读取图片素材，首先显示处于封面状态的初始窗口。

2\. 解锁菜单：在封面任意位置单击鼠标左键或敲击键盘，页面将切换至“难度选择与读档”菜单界面。

3\. 配置与初始化：

&#x20;  1)新建游戏：点击“初级”、“中级”或“高级”按钮，系统将根据设定的网格尺寸和地雷数量重新初始化并随机布雷，随后进入战场界面。

&#x20;  2)恢复进度：点击“加载存档”，系统自动读取同录下的 "save.dat" 文件恢复上一回的棋盘及游戏进度。

4\. 游玩操作核心环节：

&#x20;  1)排雷：鼠标左键点击灰色盲区区块。若安全且周围无雷，系统将沿四周无接触边界展开大面积判定；若踩中地雷则立刻触发失败状态。

&#x20;  2)标记：鼠标右键点击怀疑有地雷的盲区以插上旗帜标记，再次右键可撤销标记。

&#x20;  3)判定胜负：当翻开所有不含地雷的格子后，系统提示游戏胜利。

5\. 周边控制：在此阶段中可随时点击右侧栏按钮：

&#x20;  1)存档：写入当前棋盘数据至文件 "save.dat"。

&#x20;  2)再来一局：清空当前状态，强制回到步骤 2 的主菜单。

&#x20;  3)退出：关闭当前窗口并终止程序进程。
